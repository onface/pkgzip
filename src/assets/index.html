pkgzip

pkgzip gives you a webpack'd bundle for your npm packages.

To get started try playing with this URL: https://pkgzip.com/bundle.js?packages=left-pad

Can I just use GoMix or Webpackbin instead?

Yes! But sometimes you might want to use pkgzip instead:

  • Other services can sometimes not work with browser-specific package functionality
  • You might want to build a static HTML page with a single <script> tag

How does this differ to unpkg.com?

The main difference is that pkgzip includes all the dependencies of your packages as well.

  • Bundles dependencies as well
  • Supports multiple packages in a single request
  • Exposes packages as via window.pkgName

But if your situation is covered by unpkg then use that, it's awesome!

Features

  • Supports multiple packages in a single request (?packages=left-pad,right-pad)
  • Supports fixed (1.9.0) and flexible (@1.x / @^1.2.0 / @latest / etc) semver ranges
  • Minification (?flags=minify)
  • Deduplication (?flags=minify,dedupe)
  • Caches results server-side for fast 2nd-time responses
  • Grabs requested packages using yarn (much faster)
